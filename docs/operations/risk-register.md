**Minor, not tracked as separate risk-register rows** (logged in `TECHNICAL_DEBT.md` instead): `scripts/ops/dev_cleanup.py`'s signal loop doesn't re-verify a PID against `/proc` immediately before `SIGTERM`/`SIGKILL` (stale-snapshot race after a 5s wait); `svos/governance/snapshot.py`'s nominally-read-only snapshot path can bootstrap registry state via `ensure_strategy(...)` on a strategy's first call; `tmp/run_verification.py` hardcodes a user-specific absolute path.
